# Story H2 — Klareres Handling für „Fehler“-Status

## Beschreibung
Bei `ERROR` soll die UI konkrete, handlungsorientierte Hinweise geben, wie der Fehler behoben werden kann (Keys, Quota, PDF zu groß, Netzwerk).

## Akzeptanzkriterien
- `ERROR` zeigt eine Liste „Nächste Schritte“ (mehr als nur generischer Text).
- Hinweise sind abhängig vom Fehlertext (z.B. 401/403/Key, 429/Quota, PDF/Size, Network/Fetch).
- Kein neues UI-Pattern und keine neue Seite; nur Detail-Modal.

## Out of Scope
- Automatisches Rotieren/Verwalten von API-Keys
- Backend-Logging/Telemetry

## Touchpoints
- [components/DetailModal.tsx](../../components/DetailModal.tsx)

## Test-Hinweise
- Manuell: Beleg in `ERROR` öffnen und unterschiedliche Fehltexte prüfen.
