# Story J1 — DATEV-kompatibler Buchungsstapel-Export (CSV)

## Beschreibung
Ein zusätzlicher Export erzeugt ein DATEV-nahes CSV-Format („Buchungsstapel“/Import-CSV), sodass die Übergabe an Steuerberater/DATEV-Import weniger manuell ist.

Fokus ist ein **minimaler Kernumfang** (nicht jedes DATEV-Feld), aber konsistent und stabil.

## Akzeptanzkriterien
- Es gibt einen Export, der pro **Dokument** einen **Buchungssatz** erzeugt.
- Der Export berücksichtigt die aktuell gefilterte Dokumentliste (Jahr/Quartal/Monat).
- Der Export enthält mindestens:
  - Belegdatum (ISO)
  - Betrag (Brutto oder Netto+Steuer je definierter Logik)
  - Sollkonto + Haben-/Gegenkonto
  - Buchungstext (z.B. Lieferant + Belegnummer)
  - Belegfeld (z.B. interne Nummer / Belegnummer)
- Dateiname enthält Filter-Tag + Timestamp.

## Out of Scope
- Vollständige DATEV-EXTF-Spezifikation / alle optionalen Felder
- Import zurück in die App
- UI-Historie/Versionierung der Exporte

## Touchpoints (voraussichtlich)
- components/DatabaseGrid.tsx (Export-Button + Download)
- services/exportPreflight.ts (ggf. DATEV-spezifische Checks)
- services/storageService.ts (optional: gemeinsame Export-Helfer)
