# K1: Accounting Field Catalog

## Overview

Complete field catalog for solar accounting OCR system with standardized field definitions, data types, validation rules, and business logic mappings.

## Field Categories

### Core Financial Fields

| Field ID | Field Name     | Data Type | Required | Validation             | Business Logic                              |
| -------- | -------------- | --------- | -------- | ---------------------- | ------------------------------------------- |
| F001     | Invoice Number | String    | Yes      | Pattern: INV-\d{6}     | Primary identifier for transaction tracking |
| F002     | Invoice Date   | Date      | Yes      | ISO 8601 format        | Used for period closing and aging analysis  |
| F003     | Due Date       | Date      | Yes      | After invoice date     | Calculates payment terms and aging          |
| F004     | Invoice Amount | Decimal   | Yes      | > 0, 2 decimals        | Total amount due, used in aging reports     |
| F005     | Tax Amount     | Decimal   | No       | >= 0, 2 decimals       | Separate tax tracking for compliance        |
| F006     | Net Amount     | Decimal   | Yes      | = Invoice Amount - Tax | Core accounting entry amount                |

## Validation Rules

### Cross-Field Validations

- Due Date must be >= Invoice Date
- Invoice Amount must equal sum of Line Items
- All line items must map to valid GL Accounts
- Net Amount must be within 0.01 of (Invoice Amount - Tax Amount)

### Business Rule Validations

- Invoices from blocked vendors must be flagged
- Amounts exceeding approval limits require escalation
- Duplicate invoices detected within 30-day window
- Tax calculations validated against configured rates
